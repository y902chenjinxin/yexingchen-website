> 版本: v2.0

> 状态: 评审中

---

## 1. 产品概述

### 1.1 一句话定位

打造"天人合一"沉浸感的修仙仙府首页，让用户感受到"进入仙门"而非"打开网页"。

### 1.2 目标用户

- 修仙/仙侠文化爱好者
- 追求高品质视觉的用户
- 对中国传统文化有认同感的访客

### 1.3 核心改变

从"展示型首页"升级为"沉浸式仙府体验"，景、物、人、声、时、空、感、叙、随机、光、字、转场、包容十四维度全方位升级。

---

## 2. 设计理念

### 2.1 天人合一哲学

```
                    [用户] ←→ [物] ←→ [景]
                       ↑         ↑
                       └─────────┘
                    形成"修仙世界观"
```

- **景（环境）**: 五层背景纵深，营造仙山氛围
- **物（物件）**: 活的灵物，有灵性的存在
- **人（用户）**: 修仙者身份，不是旁观者

### 2.2 设计原则

| 原则 | 说明 |
|------|------|
| **有根有据** | 每样东西出现要有"修仙逻辑"，不能堆砌 |
| **动则有意** | 所有动画要暗示"灵气"，不是机械运动 |
| **层次分明** | 景在最远、物在中、人（用户）最近 |
| **相互呼应** | 三者不是独立的，是联动的 |
| **包容可达** | 每个功能必须可关闭、可降级 |

### 2.3 核心设计哲学

> **有5%用户用最高配置欣赏，95%用户用最稳配置存在。**
>
> **天人合一的最高境界不是"花里胡哨"，而是"每个用户都能在自己设备上感受到的那份恰到好处"。**

## 3. 完整功能体系

### 3.1 【景】五层背景纵深

**描述**: 仙侠场景的层次感，像西方油画透视一样讲究

| 层 | 名称 | 视觉效果 | 移动速度 |
|----|------|---------|---------|
| Layer 1 | 天穹深处 | 深墨色+微弱星点，最远的背景 | 几乎静止 |
| Layer 2 | 远山如黛 | 淡墨山脉剪影，虚淡如烟 | 极慢 |
| Layer 3 | 灵气涌动 | 半透明渐变，有呼吸感的流动 | 慢 |
| Layer 4 | 中景云海 | 清晰云层，有层次感 | 中速 |
| Layer 5 | 近景薄雾 | 最接近用户的层，轻纱般飘动 | 较快 |

**CSS变量定义**:
```css
--layer-1-opacity: 0.3;
--layer-2-opacity: 0.5;
--layer-3-opacity: 0.7;
--layer-4-opacity: 0.9;
--layer-5-opacity: 0.4;

--layer-1-speed: 0;
--layer-2-speed: 0.5s;
--layer-3-speed: 1s;
--layer-4-speed: 2s;
--layer-5-speed: 3s;
```

**实现方式**: CSS animation + CSS variables，parallax效果

### 3.2 【景】墨韵留白

**描述**: "留白"不是空白而是"道"。内容块之间用1.6黄金比例间距，呼吸节奏以4-7-8调息为基准

**动画时序**:
- 4s 渐显
- 7s 持续
- 8s 渐隐

**CSS变量**:
```css
--breath-in: 4s;
--breath-hold: 7s;
--breath-out: 8s;
--breath-total: 19s;
```

---

### 3.3 【景】丁达尔光柱

**描述**: 浮空岛上方有3-5道斜射的金色光柱，光柱内有缓慢飘动的尘埃粒子

**规格**:
- 光柱数量: 3-5道
- 颜色: `var(--color-gold-light)`, opacity 0.3-0.6
- 粒子: 遵循种子随机，避免不连贯

**CSS变量**:
```css
--god-ray-color: rgba(201, 169, 98, 0.4);
--god-ray-count: 4;
--god-ray-particle-opacity: 0.5;
--god-ray-animation-duration: 8s;
```

### 3.7 【人】道号体系

**描述**: 首次访问时给用户随机生成"道号"（如"青莲居士"、"听雨真人"）

**实现**:
- 道号库: 50+古风道号
- 存储: localStorage
- 显示位置: 右上角用户区
- hover显示: "修行于天外天·第X日"

**道号列表(示例)**:
```
青莲居士 | 听雨真人 | 凌霄剑仙 | 墨染书生 | 逍遥子
云中君 | 碧落仙 | 丹青客 | 问道者 | 归云溪
```

### 3.8 【人】修为进度可视化

**描述**: 访问者的停留时长、点击次数、滚动深度转化为"修为"

**实现**:
- 存储: localStorage
- 显示位置: 页面右下角，小篆印章风格
- 格式: "今日修为+12"

**计算规则**:
- 停留10分钟 = +1修为
- 点击5次 = +1修为
- 滚动到底 = +1修为

### 3.9 【人】鼠标灵气轨迹

**描述**: 鼠标移动时，轨迹留下淡金色粒子

**规格**:
- 生命周期: 1.2s
- 最大同时存在: 30个粒子
- 渲染: Canvas
- 效果: 与背景岛屿产生"隔空引物"的视觉关联

### 3.10 【声】声景合一

**描述**: 听觉设计，让修仙不只靠眼睛

| 声音层 | 来源 | 效果 |
|--------|------|------|
| 环境BGM | 古琴/笛/埙 | 随时间变化（子时=空灵、午时=明亮）|
| 岛屿音效 | hover触发 | 音乐岛=琴音、小说岛=翻书、工具岛=机关声 |
| 操作反馈音 | 点击/滑动 | 灵气碰撞的清脆声 |
| 氛围音 | 持续播放 | 风声、水流、云涌 |

**关键技术**: hover岛屿时短暂触发音效，不是强行播放，是"灵气共鸣"

---

### 3.11 【声】音频驱动视觉

**描述**: BGM通过AnalyserNode提取频率数据，岛屿亮度、粒子密度与音乐联动

**映射规则**:
- FM频段（高频）→ 粒子密度
- AM频段（低频）→ 岛屿呼吸强度

**降级**: 关掉音乐后自动回退到基础动效

### 3.25 【转场】砚台蓄墨转场

**描述**: 页面切换不是简单fade，而是"砚台蓄墨→一笔落定"

**时序**:
1. 屏幕四角墨色从无到有凝聚
2. 汇聚为一条线
3. 横向展开为新页面

**CSS**:
```css
--ink-transition: cubic-bezier(0.65, 0, 0.35, 1);
--ink-duration: 0.8s;
--ink-color: var(--color-ink-deep);
```

## 5. 技术实现要点

### 5.1 性能保障

- **视图分块懒加载**: IntersectionObserver + requestIdleCallback
- **OffscreenCanvas预渲染**: 粒子/噪点用Worker
- **Shader降级链**: WebGL2 → Canvas2D → CSS渐变
- **GPU加速**: 动画用transform而非top/left

### 5.2 降级策略

每个功能都必须有降级路径：
- 粒子系统 → 静态装饰
- 音频驱动 → 基础动画
- 手势道法 → 普通点击

### 5.3 CSS变量体系

所有颜色必须通过CSS变量引用，禁止硬编码：

```css
:root {
  /* 核心色系 */
  --color-ink-deep: #0A0C12;
  --color-gold-primary: #C9A86C;
  --color-jade: #4A7C59;

  /* 岛屿专属色 */
  --island-music: #9B8DC9;
  --island-novel: #D4C4A8;
  --island-video: #A87C9C;
  --island-log: #7A9B7C;
  --island-tool: #C49A6C;

  /* 层次透明度 */
  --layer-1: 0.3;
  --layer-2: 0.5;
  --layer-3: 0.7;
  --layer-4: 0.9;
  --layer-5: 0.4;
}
```
