# PRD v1.6.2 - 安全与体验优化

> **版本**: v1.6.2
> **日期**: 2026-05-29
> **目标**: 修复P1级安全与体验问题，完善CI自动化

---

## 一、P1 级（本周修复）

| ID | 问题 | 类别 | 解决方案 |
|----|------|------|----------|
| P1-001 | 验证码6位纯数字可暴力破解 | 安全 | 验证码增加复杂度要求（字母+数字混合，4位以上），或接入滑动验证/图片验证 |
| P1-002 | 权限校验仅依赖JWT payload | 安全 | 后端每次请求增加数据库角色校验，杜绝前端伪造权限 |
| P1-003 | Token存localStorage无过期检查 | 前端 | 前端增加Token过期检查，过期则静默刷新或跳转登录 |
| P1-004 | Math.random()导致粒子动画不连贯 | 前端 | 改用伪随机数生成器（seeded RNG）或统一粒子初始化逻辑 |
| P1-005 | npm audit/pip audit未自动化 | CI | 将安全审计加入CI，失败则阻止合并 |
| P1-006 | 无统一需求池 | PM | 已完成（docs/ISSUES.md） |
| P1-007 | 复盘Action Items无追踪 | PM | 建立Action Items追踪机制（GitHub Project或线性表格） |

---

## 二、P2 级（下版本规划）

| ID | 问题 | 类别 | 解决方案 |
|----|------|------|----------|
| P2-001 | 无API版本控制 | 后端 | 引入API版本前缀 `/api/v1/`、`/api/v2/`，旧版本兼容过渡 |
| P2-002 | 错误码体系不统一 | 后端 | 建立全局错误码枚举，统一错误响应格式 |
| P2-003 | 数据库无连接池配置 | 后端 | SQLAlchemy配置连接池参数（pool_size、max_overflow） |
| P2-004 | 无skip link键盘导航 | 前端 | 增加"跳转到内容"等skip link，支持键盘导航 |
| P2-005 | 备份无异地冗余 | 部署 | 配置异地备份（异地COS存储或定期导出） |
| P2-006 | 无部署监控告警 | 部署 | 配置PM2监控 + 业务健康检查告警 |
